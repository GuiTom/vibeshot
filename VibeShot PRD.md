## 1. 产品概述

### 1.1 产品定位

VibeShot 是一款专注于男性个人形象照片的 AI 生成平台，定位为"AI 摄影师 + AI 形象顾问"。它不是简单的 AI 换脸工具，而是通过姿势智能（Pose Intelligence）、场景模板和风格化处理，帮助男性用户生成高质量社交照片。

**核心价值主张**：让每一个"不会拍照"的男生，都能拥有让人眼前一亮的人物照片。

### 1.2 目标用户

| 用户画像 | 特征描述 |
|---------|---------|
| 社交头像用户 | 微信、微博、小红书等社交平台头像，需要专业感照片提升个人形象 |
| 交友软件用户 | Tinder、Bumble、探探、Soul 等平台，需要高质量照片提升匹配率 |
| 内容创作者 | 小红书、Instagram、B站 UP 主，需要统一风格的个人形象照 |
| 职场形象需求者 | LinkedIn、简历、职场社交，需要专业形象照展示个人品牌 |
| 穿搭预览用户 | 希望预览不同穿搭风格在新场景下的效果 |

### 1.3 市场机会与痛点

#### 现有男性用户的核心痛点

- **不会摆 pose**：面对镜头僵硬，不知道什么姿势自然好看
- **不知道什么角度帅**：缺乏镜头感，拍出来尴尬
- **没有专业设备**：手机自拍效果差，光线、构图都不理想
- **不会营造氛围感**：照片缺乏故事性和吸引力
- **现有解决方案不足**：AI 换脸产品同质化严重，缺少真正懂男性的产品

#### 市场机会

- 男性社交照片市场巨大且未被充分服务
- Tinder/Bumble 用户愿意为高质量照片付费（回报直接可见）
- 小红书/Instagram 等平台对优质形象内容需求旺盛
- 相比纯换脸工具，具有"实用价值"的产品更有长期生命力

### 1.4 核心差异化

VibeShot 的核心差异化在于 **Pose Intelligence（姿势智能）**——不是简单的 AI 生图，而是将专业摄影知识产品化：

- 什么姿势适合什么脸型
- 如何显高、显肩宽
- 如何营造少年感 vs CEO 感
- 每种风格绑定 pose + 镜头焦距 + 表情 + 手势 + 光线 + 构图

## 2. 核心功能模块

### 2.1 人物一致性系统（最核心）

用户上传 3~10 张不同角度、表情的照片，AI 学习用户面部及体型特征后，可持续生成保持一致性的多样化照片。

#### 2.1.1 照片上传规范

| 类型 | 说明 |
|-----|------|
| 正脸照 | 光线均匀，正面视角，表情自然（必选 1 张） |
| 半身照 | 胸部以下视角，展示体型轮廓（可选，AI 将按标准体型生成） |
| 侧脸照 | 左右各一张，展示面部立体感（可选） |
| 全身照 | 可选，帮助 AI 更好理解体型比例（可选） |

#### 2.1.2 AI 学习与生成

- 自动检测照片质量，过滤模糊/过度美颜照片
- 基于 LoRA/IPAdapter 技术学习用户特征
- **对于缺失的照片（如侧脸、半身），AI 将根据标准男性体型生成**
- 生成结果保持面部相似度 >85%，体型比例一致
- 生成时间：单张 30~60 秒（取决于服务器负载）

### 2.2 姿势库系统（核心产品壁垒）

姿势库是 VibeShot 与其他 AI 生图产品的本质区别。每个风格维度都经过专业摄影师调研和数据化，形成可配置的参数组合。

#### 2.2.1 姿势风格分类体系

| 分类 | 男生风格 | 女生风格 |
|-----|---------|---------|
| 冷淡感 | ✓ | ✓ |
| 松弛感 | ✓ | ✓ |
| 精英感 | ✓ | ✓ |
| 痞帅 | ✓ | — |
| 阳光 | ✓ | ✓ |
| 韩系 | ✓ | ✓ |
| 日系 | ✓ | ✓ |
| 千金感 | — | ✓ |
| 清冷 | — | ✓ |
| 甜妹 | — | ✓ |
| 御姐 | — | ✓ |

#### 2.2.2 风格参数绑定

每个风格绑定以下 6 个维度的参数：

1. **Pose（姿势）**：标准站姿/坐姿/靠姿/走姿/回头看等
2. **镜头焦距**：35mm（环境人像）、50mm（标准）、85mm（肖像）、135mm（空气感）
3. **表情**：自然微笑、酷感、沉思、轻松大笑等
4. **手势**：插兜、抱臂、自然下垂、摸头发、手指比划等
5. **光线**：侧光、逆光、蝴蝶光、自然散射光、戏剧光
6. **构图**：三分法、中心构图、对角线、框架构图

#### 2.2.3 姿势智能推荐

用户上传照片后，AI 分析用户特征（脸型、体型、气质），推荐最适合的风格组合：

- **圆脸推荐**：侧脸姿势 + 85mm 焦距，视觉拉长效果
- **矮个子推荐**：仰拍角度 + 高腰构图 + 全身站立姿势
- **窄肩推荐**：抱臂姿势 + 35mm 环境人像 + 侧光
- **少年感**：自然表情 + 日系光线 + 随机抓拍姿势
- **CEO 感**：正装 + 中心构图 + 85mm + 侧光

### 2.3 场景模板

预置高质量场景模板，用户只需点选即可应用。场景覆盖主流社交媒体需求。

#### 2.3.1 场景分类

| 场景名称 | 适用场景描述 |
|---------|------------|
| 精品咖啡馆 | 休闲时尚、小红书风格、日常感 |
| 写字楼/都市天际线 | 职场精英感、城市奋斗感 |
| 夜景街头 | 氛围感、神秘感、电影感 |
| 健身房/运动 | 健康活力、运动型人设 |
| 豪华跑车旁 | 高端生活方式、展示品味 |
| 东京/首尔街景 | 韩系/日系风格、旅行感 |
| 樱花树下 | 浪漫氛围、春季限定风格 |
| 高级酒店大堂 | 奢华感、旅行度假风格 |
| Apple Store | 科技感、极简风格 |
| 户外自然/海边 | 阳光活力、旅行探索风格 |
| 纯色/简约背景 | 通用头像、证件照风格、职场形象照 |

### 2.4 智能推荐功能

这是最容易病毒传播的功能——"把我变成会拍照的人"。

#### 2.4.1 功能描述

用户上传一张普通自拍，AI 输出完整的个人形象改造建议：

- 10 张最适合他的姿势参考图（从姿势库匹配）
- 最适合他的穿搭风格建议
- 最适合他的发型方向
- 最适合他的镜头角度和光线方案

#### 2.4.2 输出形式

- AI 生成的参考图 + 文字建议报告（PDF/图片格式）
- 可一键应用推荐方案生成照片
- 建议附带"形象升级分"——0~100 分，量化提升空间

### 2.5 输出管理与分享

- 照片九宫格预览：自动生成适合朋友圈/小红书的排版
- 一键下载：原图/压缩图/多种尺寸
- 社交平台直接分享：微信、小红书、Instagram
- 照片收藏夹：历史生成记录管理
- 照片删除：支持删除不希望保留的生成结果

## 3. 技术架构

### 3.1 整体架构概览

采用前后端分离架构，AI 服务独立部署，Java 与 Python 通过 REST API 解耦。

#### 3.1.1 技术选型总览

| 层级 | 技术栈 | 说明 |
|-----|-------|------|
| 前端 | Flutter / Vue3 + Capacitor | 跨平台移动应用（iOS/Android） |
| Web 前端 | Vue 3 + Vite + TailwindCSS | MVP 优先 Web 版 |
| 后端服务 | Spring Boot + PostgreSQL + Redis | Java 团队核心能力区 |
| 文件存储 | MinIO（S3 兼容） | 对象存储，兼容 AWS S3 API |
| AI 服务 | Python: ComfyUI / Diffusers / Flux / SDXL | 独立部署，REST API 供 Java 调用 |
| 支付 | Stripe（海外）/ 微信支付/支付宝 | 订阅制 + 按次付费 |
| 部署 | Docker + Kubernetes / 腾讯云 | 弹性扩缩容 |

### 3.2 AI 服务层

#### 3.2.1 人物一致性方案

MVP 阶段推荐 OpenAI Images API（简单稳定），后续迁移至自部署 Flux + LoRA 方案降低成本。

| 方案 | 优点 | 缺点 |
|-----|------|-----|
| 方案 A: Flux + LoRA | 成本低、可自部署、出图质量高 | 部署复杂、LoRA 训练需 GPU |
| 方案 B: OpenAI API | 简单、稳定、API 成熟 | 成本较高、一致性可控性稍弱 |
| 推荐方案: IPAdapter+InstantID | 人物一致性优秀、轻量级、不需 LoRA | 需 SDXL 基础模型、推理速度中等 |

#### 3.2.2 AI 能力模块拆分

- **人脸特征提取**：提取用户面部关键特征向量
- **姿势推荐引擎**：基于用户照片匹配最佳姿势参数
- **图像生成引擎**：基于 Flux/SDXL + 场景 + 姿势生成照片
- **图像质量评估**：检测生成结果是否达标（清晰度、五官完整性）
- **智能建议生成**：分析用户特征，生成形象改造报告

### 3.3 后端服务设计

#### 3.3.1 核心 API

| API | 方法 | 说明 |
|-----|------|------|
| `/api/v1/user/register` | POST | 用户注册/登录 |
| `/api/v1/avatar/upload` | POST | 上传用户照片构建人脸特征 |
| `/api/v1/generate/photo` | POST | 生成照片（风格+场景+姿势） |
| `/api/v1/pose/recommend` | POST | 智能推荐姿势和风格 |
| `/api/v1/photo/history` | GET | 获取历史生成记录 |
| `/api/v1/order/create` | POST | 创建付费订单 |
| `/api/v1/subscription/status` | GET | 查询订阅状态和剩余次数 |

#### 3.3.2 数据模型

- **User**：用户信息、订阅状态、积分余额
- **UserAvatar**：用户上传的照片及提取的特征向量
- **Style**：姿势风格定义（风格类型 + 参数组合）
- **Scene**：场景模板定义（场景名称 + 背景图 + 光线参数）
- **GeneratedPhoto**：生成记录（用户 ID + 风格 ID + 场景 ID + 生成时间 + 图片 URL）
- **Order**：订单记录（订单号 + 用户 ID + 套餐类型 + 支付状态 + 金额）
- **Subscription**：订阅记录（订阅类型 + 开始/结束时间 + 使用次数）

## 4. 商业模式与定价

### 4.1 核心商业模式

VibeShot 的商业模式核心不是卖"生成次数"，而是服务男性用户在社交场景中的形象提升需求——这是一个回报直接的付费动机。

### 4.2 定价策略

#### 4.2.1 订阅制（主流）

| 套餐 | 价格 | 生成次数 | 功能范围 |
|-----|------|---------|---------|
| 免费体验 | 免费 | 3 张 | 基础风格 + 3 个场景 |
| 月度订阅 | ¥39/月 | 50 张/月 | 全风格 + 全场景 + 姿势推荐 |
| 年度订阅（推荐） | ¥299/年 | 600 张/年 | 全部功能 + 优先队列 + 智能推荐报告 |
| 按次付费 | ¥3/张 | 无限制 | 适合低频用户 |

#### 4.2.2 增值服务

- **形象升级报告**：¥9.9/次，生成完整个人形象改造方案（含姿势、穿搭、发型建议）
- **高清原图下载**：免费用户仅支持压缩图，订阅用户可下载 4K 原图
- **批量生成包**：¥19.9，10 张同风格批量生成

### 4.3 变现路径

- **阶段一（0~3 个月）**：Web 版 MVP，通过小红书/TikTok 引流，验证产品 PMF
- **阶段二（3~6 个月）**：推出 App，接入支付，实现月流水突破 ¥10 万
- **阶段三（6~12 个月）**：推出企业版（B2B），为婚恋平台、猎头公司提供 API 服务
- **阶段四（12 个月+）**：社区化，推出男性形象社区，内容+工具+社交闭环

## 5. MVP 产品规划

### 5.1 MVP 定义

第一版 MVP 专注于验证核心价值闭环：**上传照片 → 生成好看的社交照片**。

#### 5.1.1 MVP 功能范围

| 功能 | MVP | 后续版本 |
|-----|-----|---------|
| 用户注册/登录 | ✓ 手机号/微信 | Apple ID / Google 登录 |
| 人脸特征上传 | ✓ 3 张照片 | 扩展至 10 张 |
| 姿势库 | ✓ 6 种男生风格 | 扩展至 12+ 风格，含女生 |
| 场景模板 | ✓ 6 个场景 | 扩展至 12+ 场景 |
| 姿势智能推荐 | ✓ 自动推荐 | 详细分析报告 |
| 照片生成 | ✓ 单张生成 | 批量生成 |
| 支付 | ✓ 微信/支付宝 | Stripe 接入（海外） |
| 输出管理 | ✓ 下载 + 分享 | 九宫格排版 + 批量下载 |

### 5.2 MVP 阶段需避免的功能

- 超复杂 AI Agent 系统
- 视频生成功能
- 实时换装
- 3D 建模
- 复杂的用户社交功能
- 多语言国际化（先专注中文市场验证）

### 5.3 优先级排序

| 优先级 | 核心功能 | 工时 |
|-------|---------|-----|
| P0 | 人脸特征上传 + OpenAI API 生成 + 姿势选择 + 下载分享 | 4 周 |
| P1 | 姿势智能推荐引擎 + 支付订阅系统 + 历史记录管理 | 2 周 |
| P2 | 形象升级报告 + 九宫格排版 + 场景扩展 | 2 周 |

## 6. 运营与增长策略

### 6.1 冷启动策略

初期以内容营销为主，无需大规模投放。通过高质量案例和反差效果吸引目标用户。

#### 6.1.1 内容营销渠道

- **小红书**：发布"男生拍照改造前后对比"系列，用真实案例展示产品效果（核心渠道）
- **抖音/TikTok**：发布"不会拍照男生 → AI 帮你变成拍照达人"短视频，容易病毒传播
- **微信视频号**：配合公众号，形成私域+公域双循环
- **Reddit（海外）/ 即刻（国内）**：垂直社区精准投放

#### 6.1.2 增长飞轮设计

1. **钩子**：免费生成 3 张照片（无需注册）
2. **转化**：生成效果超出预期 → 用户愿意付费获取更多
3. **传播**：生成好看的照片 → 用户分享到社交平台 → 形成自然流量
4. **留存**：订阅制 + 历史记录 + 个性化推荐 → 长期使用习惯

### 6.2 关键指标（KPI）

| 指标 | 目标值 | 衡量周期 |
|-----|-------|---------|
| 注册转化率（免费 → 付费） | > 5% | 按月 |
| 月活跃付费用户（MPU） | MVP 首月 > 100 | 按月 |
| 用户 LTV（生命周期价值） | > ¥100 | 按季度 |
| 单张生成成本 | < ¥0.5 | 按月 |
| NPS（用户推荐值） | > 50 | 按季度 |
| 自然流量占比 | > 30%（6 个月后） | 按月 |

## 7. 风险与规避策略

| 风险类型 | 风险描述 | 规避策略 |
|---------|---------|---------|
| 技术风险 | AI 生成质量不稳定，人物相似度不够 | MVP 使用 OpenAI 官方 API，确保稳定性；后续逐步引入 LoRA 定制 |
| 合规风险 | AI 生成照片用于交友可能涉及平台违规（如 Tinder） | 产品定位为"形象提升参考"，非伪造身份；遵守各平台 TOS |
| 竞争风险 | 大厂（如字节、腾讯）入场，姿势库被复制 | 姿势库持续迭代，构建数据壁垒；社区化运营，建立用户粘性 |
| 成本风险 | AI 算力成本高，用户增长后毛利下降 | 早期用 OpenAI API 验证 PMF；后期自部署 Flux + LoRA 降至 <¥0.3/张 |
| 隐私风险 | 用户上传照片涉及隐私，需安全存储和处理 | 本地特征提取后删除原图，仅存储特征向量；符合 GDPR/个保法 |

### 9.1 术语表

| 术语 | 定义 |
|-----|------|
| **Pose Intelligence** | 姿势智能，VibeShot 的核心差异化能力——通过 AI 分析用户特征，匹配最适合的姿势、光线、构图参数组合 |

| **InstantID** | 一种基于面部关键点的身份保持技术，可在保持高相似度的同时改变姿势/场景 |
| **MPU** | Monthly Paying Users，月度付费用户数 |
| **LTV** | Lifetime Value，用户生命周期总价值 |
| **NPS** | Net Promoter Score，净推荐值，衡量用户忠诚度和口碑传播能力 |
